# KIVI：KV 缓存混合量化

KIVI 通过结合通道量化和令牌量化，在保证模型性能的同时大幅降低 KV 缓存的内存占用。

## 1. 通道量化的键缓存 (Channel-Wise Key Quantization)

将键（Key）按照通道（channel）进行量化。每个通道对应不同的特征维度，通道量化对每个通道单独计算缩放因子和零点，保留通道间的独立性。

- 减少内存占用，尤其适合大规模模型
- 保证通道间独立性，避免全局量化导致的信息丢失
- 适用于注意力机制中需要频繁访问键的场景

## 2. 令牌量化的值缓存 (Token-Wise Value Quantization)

将值（Value）按照令牌（token）进行量化。每个令牌对应一个值向量，量化粒度以令牌为单位。

- 精细化的量化方式，尽量保留每个令牌的语义信息
- 对动态生成的令牌，量化后的值缓存可加速后续计算
- 适用于生成式模型中需要处理大量令牌的场景

## 3. 混合策略

KIVI 的核心在于 Key 和 Value 采用不同的量化维度：
- Key 按通道量化：解决通道间分布差异问题
- Value 按令牌量化：适应动态序列特性

两者协同优化显存与精度，特别适合 Transformer 架构中的注意力机制。
