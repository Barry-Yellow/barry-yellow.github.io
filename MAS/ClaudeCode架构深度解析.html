全局架构鸟瞰

Claude Code 的本质不是一个聊天应用，而是一个面向工程任务的多智能体可扩展执行内核

🏗️ 设计哲学

  • ① 统一Runtime，分层角色——主Agent/子Agent/teammate共享同一个query循环
  • ② 上下文工程 > 功能堆砌——缓存命中率、压缩策略、分流落盘是一等公民
  • ③ 工具调用事务化——每次调用经过schema→验证→hook→权限→执行→遥测
  • ④ 文件系统是状态底座——transcript/memory/plan/team全部文件化持久化
  • ⑤ 多Agent有治理结构——leader-worker权限同步、显式消息、共享任务板

⚡ 两种会话引擎

引擎入口场景
交互式引擎main.tsx → Ink TUI用户在终端交互
SDK引擎QueryEngine.ts程序化调用、headless模式

两种引擎最终都调用同一个 query() 函数——这就是"统一Runtime"的体现

数据流与通信方式

模块间如何通信和传递数据

1️⃣ 消息流（核心数据通道）

所有模块通过 Message[] 数组通信

UserMessage → 用户输入
AssistantMessage → 模型输出（text + tool_use）
SystemMessage → 系统指令
AttachmentMessage → 文件附件
ProgressMessage → 进度通知
SystemCompactBoundaryMessage → 压缩边界

2️⃣ AppState（全局状态总线）

React Context + 不可变Store模式

写入：store.setState() 产生新不可变状态
读取：useSyncExternalStore 订阅
跨组件：AppStateProvider 包裹整个组件树

3️⃣ Mailbox（跨Agent邮箱）

跨Agent异步消息传递

• Leader → Worker：指令下发、权限响应

• Worker → Leader：结果上报、权限请求

• 基于 React Context + 事件机制

4️⃣ 文件系统（持久化通道）

设计巧思总结

Claude Code 架构中的关键设计决策

🔄

统一Runtime，分层角色

主Agent/子Agent/teammate共享同一个query循环，避免重复实现

🧠

上下文工程 > 功能堆砌

缓存命中率、压缩策略、分流落盘是一等公民

🔒

工具调用事务化

每次调用经过schema→验证→hook→权限→执行→遥测完整管线

💾

文件系统是状态底座

transcript/memory/plan/team全部文件化持久化，崩溃可恢复

🏗️

多Agent有治理结构

leader-worker权限同步、显式消息、共享任务板

⚡

延迟导入+死代码消除

bun:bundle feature门控 + 动态import，快速路径零加载

🔁

递归生成器模式

query() 是async generator，SDK可流式获取中间结果

🚀

流式工具预执行

StreamingToolExecutor不等模型输出完毕就启动读型工具

📝

Prompt分布式维护

每个工具目录下有自己的prompt.ts，就近维护易演进

✂️

缓存边界切割

静态段跨请求共享prompt cache，动态段每次重新生成

🔮

推测执行

空闲时预生成下一轮可能的响应，减少用户等待

🪝

Hook生命周期工程

标准化的拦截点，支持阻断、修改、追加上下文
