## 二、记忆与上下文管理

### 三类记忆对比

| 类型 | 实现 | 特点 |
|------|------|------|
| 上下文（短期） | 对话历史 window | 有限长度，随轮次滚动 |
| 工作记忆 | 内存缓存 / KV Store | 会话内持久，跨轮可用 |
| 长期记忆 | 向量数据库 / 知识图谱 | 跨会话，需检索 |

### 上下文过长处理方法

```
截断（最简单）→ 滑动窗口 → 关键信息提取 → Summarization
→ 递归压缩（MEM1）→ 知识图谱存储
```

**子任务上下文保持**：注入全局目标摘要 + 关键约束到每个子任务 Prompt

## 三、RAG 全链路

### 完整流程

```
文档处理 → 分块(Chunk) → 向量化(Embed) → 存入向量库
→ Query → 检索 → Rerank → 增强生成
```

### 分块策略

| 场景 | 策略 |
|------|------|
| 普通文本 | 固定大小 + Overlap |
| 代码/论文 | 按语义单元（函数/章节） |
| 表格/图片 | OCR + 多模态理解 |

### 检索质量提升

| 方法 | 说明 |
|------|------|
| Query Rewriting | LLM 改写原始 Query |
| Multi Query | 生成多个变体 Query 并行检索 |
| HyDE | 先生成假设性答案再检索 |
| Hybrid Search | BM25（关键词）+ 向量（语义）融合 |
| Rerank | Cross-Encoder 精排，关注直接相关性 |

**Hybrid Search 分数融合**：Min-Max 归一化 → 加权求和

## 四、工具调用与规划

### 工具报错重试设计

```
工具报错 → Prompt 要求模型分析错误
→ 调整参数 / 尝试替代方案
→ 重试循环（设最大次数限制）
```

### 模糊 Query 处理

- 置信度高 → 直接检索
- 置信度低 → 主动反问澄清意图

### 多工具冲突解决

| 方案 | 适用场景 |
|------|---------|
| 优先级定义 | 工具间有明确依赖 |
| 互斥锁 | 资源竞争场景 |
| Orchestrator Agent | 复杂多工具编排 |

### MCP vs A2A

| 协议 | 定位 |
|------|------|
| MCP | 模型 ↔ 外部工具/数据 标准化连接 |
| A2A | Agent ↔ Agent 通信与任务委托 |

## 五、Agent 架构设计

### 单 Agent vs 多 Agent

| 维度 | 单 Agent | 多 Agent |
|------|---------|---------|
| 复杂度 | 低 | 高 |
| 适用场景 | 简单线性任务 | 复杂解耦/并行任务 |
| 通信开销 | 无 | 消息队列 / 共享状态 / A2A |

### Router 选型

| 方案 | 优点 | 缺点 |
|------|------|------|
| LLM Router | 灵活，处理复杂情况 | 成本高、慢、不稳定 |
| 固定规则 | 快、稳定、可解释 | 灵活性差 |

### Skill 过多时的处理

```
动态加载（Tool Wrapper）→ Skill 索引 + 路由
→ 渐进式披露（按需展示 Skill 描述）
```

## 六、系统设计与工程实践

### 多级缓存策略

```
L1: 内存（热点 LLM 响应）
L2: Redis（工具调用结果 / 会话状态）
L3: Semantic Cache（语义相似 Query 复用）
```

过期策略：TTL + LRU 淘汰

### 成本 vs 速度平衡

| 手段 | 效果 |
|------|------|
| 动态模型选择 | 简单任务用小模型 |
| 流式响应 | 降低感知延迟 |
| 异步处理 | 提升吞吐 |
| 预算 + 降级 | 超限自动切换策略 |

### Agent 评估指标

| 指标 | 说明 |
|------|------|
| 任务成功率 | 最终目标达成 |
| 目标达成度 | 里程碑推进比例 |
| 自我纠错率 | 错误后自主恢复次数 |
| 工具调用准确率 | 正确调用 / 总调用 |
| 幻觉动作率 | 无效/虚构工具调用占比 |
| Token 消耗 / 延迟 | 成本与性能 |

## 七、模型微调

### LoRA / QLoRA

| 要点 | 内容 |
|------|------|
| 核心思想 | 冻结原模型，插入低秩适配器 ΔW = A·B |
| 矩阵初始化 | A：随机初始化；B：全零初始化 |
| QLoRA 额外 | 基座模型量化（4-bit NF4），节省显存 |

### DPO vs PPO

| 维度 | PPO | DPO |
|------|-----|-----|
| 奖励模型 | 需要单独训练 | 不需要 |
| 稳定性 | 较低 | 较高 |
| 数据格式 | (Prompt, Response, Reward) | (Prompt, Chosen, Rejected) |

### 垂域微调后通用能力下降

- 混合通用指令数据（数据层）
- 多任务联合训练（训练层）
- 损失函数加 KL 散度约束（目标层）
