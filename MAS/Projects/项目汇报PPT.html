2核心方法 — 根因定位 · 边界识别 · 修复动作

根因定位

Sroot(v) = α·A(v) + β·I(v) + γ·D(v) + δ·R(v)
分量含义
A(v)异常强度 — 硬信号优先
I(v)影响强度 — 可达受影响节点数
D(v)偏离程度 — 与目标偏差
R(v)修复收益 — 修复价值

定位"修复价值最高的根因点"，而非"最显眼的错误点"

污染边界识别

P(v) = maxu∈Pred(v) P(u) · w(u,v) · λ

衰减传播模型，λ = 0.85，按边权重区分真实依赖 vs 时序相邻

节点分类复用评分处理
Clean≥ 80PRESERVE
Partial60~80PATCH
Invalid< 60重跑/重写

3增量重执行 · 一致性验证 · 技术栈

增量重执行三能力

Checkpoint 恢复 — 从最靠近根因的快照恢复
Journal Replay — 未修改的工具结果优先回放旧结果
依赖注入（ADF） — 只注入依赖前沿，减少长上下文干扰

一致性验证

局部一致性 — 接口兼容 · Artifact存在性 · 工具结果匹配 · 矛盾检测
全局一致性 — 任务成功标准 · 里程碑前进 · 新错误检测 · 目标兼容
Milestone Forward：关注修复是否"推动任务向前"，而非仅"减少错误"

2Agentic RAG · 三层记忆 · 反爬策略

Agentic RAG Pipeline

Q 查询理解 → LLM改写
R1 向量检索 → 语义匹配
R2 BM25检索 → 关键词匹配
F RRF融合 → 兼顾两路
G 生成回答 → LLM评估

vs 传统RAG：Agent自主决策检索策略，不足时重检索

三层记忆管理

⚡ 短期记忆 — 内存dict · 当前会话 · TTL淘汰
💾 长期记忆 — 本地JSON · 跨会话偏好 · FIFO淘汰
🗄️ RAG记忆 — 向量数据库 · 文档知识库 · 语义检索

优先级：短期 > 长期 > RAG

反爬三层策略

🎭 行为层 — 随机延时 · 模拟人类 · 鼠标轨迹
🔍 特征层 — playwright-stealth · TLS/Canvas指纹
⏱️ 频率层 — 请求限速 · 代理轮换 · 会话间隔

3容错机制 · 技术栈 · 核心创新

四层容错体系

L1
自动重试 — 指数退避 1s→2s→4s，最多3次
L2
降级策略 — LLM Provider切换 / 采集降级
L3
状态回滚 — GlobalState快照回滚，防脏状态
L4
用户兜底 — 友好错误提示 + 建议操作

GlobalState（TypedDict）

messages user_input intent intent_detail crawler_output summary_output analysis_output final_response context error

每个Agent只修改自己负责的字段，TypedDict类型约束保证状态不可变性

核心创新点

  • 意图驱动路由 — 5种意图自动识别分发
  • Agentic RAG — 混合检索+RRF融合，Agent自主决策
  • 三层记忆 — 短期/长期/RAG分层管理
  • 多层容错 — 重试→降级→回滚→兜底
